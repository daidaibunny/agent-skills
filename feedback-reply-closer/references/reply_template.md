# Reply Draft Template

## Structure
1. Acknowledge issue.
2. State action taken.
3. Mention expected user result.
4. Invite follow-up if issue persists.

## Example
"已收到你的反馈，我们已经完成修复并发布。请刷新后重试；如果问题仍然存在，请直接回复这条消息，我们会继续跟进。"

## Guardrails
- Keep response concise and specific.
- Avoid promising uncertain timelines.
- Do not mention internal-only debugging details.
